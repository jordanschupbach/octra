In this repo, c++ is bound to a bunch of different languages using swig. Nix is
used to setup the build and default install environments. I'd like to add as
much code quality checks as possible and for every language. This includes
things like linters, formatters, and static analysis tools. The goal is to
ensure that the code is clean, consistent, and free of common mistakes across
all the languages used in the project. This will help maintain a high standard
of code quality and make it easier for developers to contribute to the project.
Additionally, I want to set up automated checks that run on every commit or
pull request to catch any issues early in the development process. This will
help prevent bugs and maintain the overall health of the codebase. Can you add
as much as possible. Make sure to use nix to manage the dependencies for these
tools and to ensure that they are. Also, I use a justfile as a command runner.
Be sure to add commands to the justfile to run these checks easily. Focus on
adding as much as possible.
